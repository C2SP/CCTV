# Accumulated ML-DSA tests

This file provides [accumulated](https://words.filippo.io/accumulated/) random
ML-DSA key generation and signature tests at various iteration counts, and
accumulated exhaustive tests for various field operations.

## Key generation and signature tests

The vectors are generated by reading seeds from one instance of SHAKE128, and
writing the derived public key and the deterministic signature of an empty
message to another instance of SHAKE128.

```
s = SHAKE128()
a = SHAKE128()
for i in range(N):
    seed = read(s, 32)
    write(a, seed_to_pk(seed))
    write(a, sign_deterministic(seed, message="", context=""))
result = read(a, 32)
```

This allows running a number of known-answer tests that would be impractical if
the results had to be stored in full.

In particular, the 60 million iteration test has a 99.9% chance of hitting any
value in [0, q). Storing that many vectors would require between 225 GB
(ML-DSA-44) and 435 GB (ML-DSA-87).

Using fixed message, context, and signature randomness leads to the same μ for
every test, but doesn't reduce coverage because μ is only hashed with either
the secret key's random K, or w1 which is derived from the random matrix A.

## Field operation tests

The vector is generated by writing to a SHAKE128 instance the following
12 values, as ASCII decimals with an optional leading `-` sign (if negative),
one per line, for each element r in ℤq from 0 to q - 1:

  - r mod± q
  - ‖r‖∞ = |r mod± q|
  - r1, r0 = Power2Round(r)
      - r1
      - r0 mod q

For ML-DSA-44 (γ₂ = (q - 1) / 88):
  - HighBits(r) = UseHint(0, r)
  - UseHint(1, r)
  - LowBits(r)
  - ‖LowBits(r)‖∞ = |LowBits(r)|

For ML-DSA-65 and ML-DSA-87 (γ₂ = (q - 1) / 32):
  - HighBits(r) = UseHint(0, r)
  - UseHint(1, r)
  - LowBits(r)
  - ‖LowBits(r)‖∞ = |LowBits(r)|

Note that HighBits(r), LowBits(r) = Decompose(r).

```
f930663417278156ab05d940294a77210a809c924d8ab63ec72f4526247602c7
```
